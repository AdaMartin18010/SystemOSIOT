## 2. ISO/IEC 15408:2022 — Common Criteria 评估方法

### 2.1 标准条款原文

ISO/IEC 15408:2022 Part 1 §4 对安全评估的总体要求（paraphrase）：

> "Evaluation is the process of assessing the IT product against the security functional requirements and assurance requirements defined in the Security Target, using evaluation criteria and evaluation methods."

Part 3 定义 EAL1–EAL7 评估保障等级（paraphrase）：

> "Evaluation Assurance Levels (EALs) provide a scale for measuring assurance, from functionally tested (EAL1) to formally verified design and tested (EAL7)."

### 2.2 与项目知识点的映射

| 标准条款 | 项目知识点 | 项目章节 |
|---|---|---|
| CC Part 1 §4（评估概念） | 安全评估、安全目标（ST）、威胁模型 | `2.2 批判分析/` |
| CC Part 3 EAL1–EAL4 | 商业级安全测试、结构化设计审查 | `2.2 批判分析/`、`2.4 形式化证明/` |
| CC Part 3 EAL5–EAL7 | 半形式化/形式化设计、验证与测试 | `2.4 形式化证明/`、`2.7 形式证明/` |
| ASE（Security Target evaluation） | 安全目标编写、安全功能要求映射 | `2.2 批判分析/` |
| ADV（Development） | 安全架构、功能规范、实现表示 | `2.3 形式化结构/` |
| ATE（Tests）/ AVA（Vulnerability assessment） | 测试覆盖、漏洞分析 | `2.7 形式证明/` |

## 3. TCSEC / Orange Book — 历史安全等级

### 3.1 标准条款原文

DoD 5200.28-STD 定义 A1–D 安全等级（paraphrase）：

> "The Trusted Computer System Evaluation Criteria (TCSEC) define a hierarchy of requirements for computer security, from minimal protection (D) through verified protection (A1), emphasizing discretionary and mandatory access control, assurance, and documentation."

### 3.2 与项目知识点的映射

| 等级 | 项目知识点 | 项目章节 |
|---|---|---|
| D（Minimal Protection） | 无可靠安全机制的系统 | `2.2 批判分析/` |
| C1/C2（Discretionary Access Control） | 自主访问控制、审计 | `2.2 批判分析/`、`2.4 形式化证明/` |
| B1/B2/B3（Mandatory Access Control） | 强制访问控制、安全标签、可信路径 | `2.2 批判分析/` |
| A1（Verified Design） | 形式化安全策略模型 | `2.4 形式化证明/`、`2.7 形式证明/` |

- `2.2 批判分析/2.2.1 主要争议.md`

## 4. NIST SP 800-53 Rev.5 — 安全与隐私控制

### 4.1 标准条款原文

NIST SP 800-53 Rev.5 控制族定义（paraphrase，来源：NIST 官方目录）：

> "AC-3 Access Enforcement: The information system enforces approved authorizations for logical access to information and system resources in accordance with applicable access control policies."
> "SC-39 Process Isolation: The information system maintains a separate execution domain for each executing process."

### 4.2 与项目知识点的映射

| 控制编号 | 控制名称 | 项目知识点 | 项目章节 |
|---|---|---|---|
| AC-2 | Account Management | 用户账户、身份鉴别 | `2.1 知识梳理/` |
| AC-3 | Access Enforcement | 访问控制矩阵、权限检查、系统调用过滤 | `2.2 批判分析/`、`2.4 形式化证明/` |
| AC-6 | Least Privilege | 最小权限原则、Capability 模型 | `2.2 批判分析/` |
| AU-6 | Audit Review | 系统调用审计、日志分析 | `2.8 系统运行时语义/` |
| CM-7 | Least Functionality | 系统调用裁剪、seccomp、Capabilities | `2.8 系统运行时语义/` |
| SC-2 | Application Partitioning | 用户态/内核态隔离、微内核 | `2.3 形式化结构/` |
| SC-39 | Process Isolation | 进程地址空间隔离、MMU/MPU | `2.3 形式化结构/` |
| SI-7 | Software, Firmware, and Information Integrity | 可信启动、代码签名、TPM 度量 | `2.4 形式化证明/` |

## 5. seL4 — 形式化验证操作系统内核

### 5.1 标准条款原文

seL4 SOSP 2009 论文核心结论（paraphrase）：

> "We present the formal, machine-checked verification of seL4: an OS microkernel proved to implement its abstract specification, preserve isolation properties between processes, and guarantee information-flow security."

### 5.2 与项目知识点的映射

| 主题 | 项目知识点 | 项目章节 |
|---|---|---|

## 6. ARM TrustZone / TPM 2.0 / UEFI Secure Boot — 可信计算根

### 6.1 标准条款原文

ARM TrustZone（paraphrase，来源 ARM Architecture Reference Manual）：

> "TrustZone technology provides a system-wide approach to security by partitioning hardware and software resources into Secure and Normal worlds."

TCG TPM 2.0 Part 1 §12 对 TCB 的描述（paraphrase）：

> "The Trusted Computing Base (TCB) is the set of hardware, firmware, and software components that are critical to the security of the system."

UEFI Spec 2.10 §32（Secure Boot，paraphrase）：

> "Secure Boot defines a platform firmware authentication process to ensure that each boot component is trusted."

### 6.2 与项目知识点的映射

| 技术 | 项目知识点 | 项目章节 |
|---|---|---|
| TrustZone Secure/Normal World | 特权级、安全态切换、可信执行环境 | `2.3 形式化结构/` |
| TPM 2.0 PCR / measured boot | 可信启动链、平台配置寄存器 | `2.4 形式化证明/` |
| UEFI Secure Boot | 启动加载器签名、密钥管理 | `2.8 系统运行时语义/` |
| ACPI / DRTM (Dynamic Root of Trust) | 运行时信任根、SMM/SEV/TXT | `2.8 系统运行时语义/` |

## 7. EAL — 评估保障等级与操作系统认证

### 7.1 标准条款原文

ISO/IEC 15408:2022 Part 3 定义 EAL（paraphrase）：

> "EAL1: functionally tested; EAL2: structurally tested; EAL3: methodically tested and checked; EAL4: methodically designed, tested and reviewed; EAL5: semiformally designed and tested; EAL6: semiformally verified design and tested; EAL7: formally verified design and tested."

### 7.2 与项目知识点的映射

| EAL | 操作系统实例 | 项目章节 |
|---|---|---|
| EAL4+ | 通用 Linux/Windows Server CC 认证 | `2.2 批判分析/` |
| EAL5–EAL6 | 高保障安全操作系统（如 INTEGRITY-178B） | `2.4 形式化证明/` |
| EAL7 | seL4（等效目标） | `2.4 形式化证明/`、`2.7 形式证明/` |
