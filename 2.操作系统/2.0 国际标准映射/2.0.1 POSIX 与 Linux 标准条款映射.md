## 2. POSIX.1-2024 §3 Definitions — 进程与线程

### 2.1 标准条款原文

POSIX.1-2024 §3.141 对 **process** 的定义（paraphrase）：

> "An address space with one or more threads executing within that address space, and the required system resources for those threads."

POSIX.1-2024 §3.283 对 **thread** 的定义（paraphrase）：

> "A single flow of control within a process. Each thread has its own thread ID, scheduling priority and policy, errno value, signal mask, and saved context."

### 2.2 与项目知识点的映射

| 标准术语 | 项目知识点 | 项目章节 |
|---|---|---|
| Process（地址空间 + 资源集合） | 进程概念、进程控制块（PCB）、进程状态机 | `2.1 知识梳理/2.1.1 基本概念.md` |
| Thread（控制流） | 线程模型、用户级/内核级线程、pthread 语义 | `2.1 知识梳理/` |
| fork/exec/wait | 进程创建、程序装入、僵尸进程、孤儿进程 | `2.1 知识梳理/`、`2.6 形式语义/` |
| Scheduling priority & policy | 调度算法、优先级调度、实时调度 | `2.3 形式化结构/`、`2.8 系统运行时语义/` |
| Signal mask & saved context | 上下文切换、信号处理、中断屏蔽 | `2.6 形式语义/`、`2.8 系统运行时语义/` |

### 2.3 证据文件

- `2.1 知识梳理/2.1.1 基本概念.md`（进程/线程定义）
- `2.3 形式化结构/2.3.2 结构模型.md`（PCB/TCB 结构）
- `2.6 形式语义/2.6.1 语义定义.md`（系统调用语义）
- `2.8 系统运行时语义/2.8.1 运行时行为与调度模型.md`（调度运行时）

### 2.4 覆盖状态

- 术语定义：**部分覆盖**
- fork/exec/wait 语义：**部分覆盖**
- 线程同步原语（pthread_mutex/cond）：**未系统映射**

### 2.5 形式化/可验证链接

- 待补充；参考方向：在 `tools/tla-specifications/` 或 `tools/coq-verification/` 建立 POSIX 进程/线程状态机模型。
- 外部参考：seL4 l4v 项目的 Isabelle/HOL 线程/调度形式化 <https://github.com/seL4/l4v>。

## 3. POSIX.1-2024 §13 — 文件系统接口

### 3.1 标准条款原文

POSIX.1-2024 §13 规定文件系统接口，包括 `open()`、`close()`、`read()`、`write()`、`lseek()`、`stat()`、`mkdir()`、`rmdir()`、`rename()` 等（paraphrase）：

> "The file system interfaces provide the means for applications to create, delete, read, write, and control files and directories, and to obtain and set file metadata."

### 3.2 与项目知识点的映射

| 标准接口 | 项目知识点 | 项目章节 |
|---|---|---|
| open/close/read/write | 文件系统抽象、文件描述符、打开文件表 | `2.1 知识梳理/` |
| lseek / file offset | 顺序/随机访问、文件位置语义 | `2.1 知识梳理/` |
| stat / metadata | 索引节点（inode）、权限、时间戳 | `2.1 知识梳理/` |
| mkdir/rmdir/rename | 目录结构、命名空间、硬链接 | `2.1 知识梳理/` |
| File permissions | 访问控制列表（ACL）、UGO 权限 | `2.2 批判分析/`、`2.4 形式化证明/` |

### 3.3 证据文件

- `2.1 知识梳理/2.1.1 基本概念.md`
- `2.3 形式化结构/2.3.2 结构模型.md`

### 3.4 覆盖状态

- 基础接口语义：**部分覆盖**
- 文件描述符表与打开文件表的形式化区分：**未覆盖**
- ACL 与 POSIX ACL 扩展：**未覆盖**

### 3.5 形式化/可验证链接

- 待补充；建议建立文件系统状态转换模型（形式化读写、创建、删除）。

## 4. POSIX.1-2024 §16 — 网络接口

### 4.1 标准条款原文

POSIX.1-2024 §16（Networking Interfaces）定义 BSD Socket API，包括 `socket()`、`bind()`、`listen()`、`accept()`、`connect()`、`send()`、`recv()` 等（paraphrase）：

> "A set of interfaces that provide interprocess communication across a network using sockets."

### 4.2 与项目知识点的映射

| 标准接口 | 项目知识点 | 项目章节 |
|---|---|---|
| socket / bind / listen / accept | 套接字抽象、服务器/客户端模型 | `2.1 知识梳理/` |
| connect / send / recv | TCP/UDP 连接、字节流/数据报 | `2.1 知识梳理/`、`2.6 形式语义/` |
| select/poll/epoll (Linux) | I/O 多路复用、事件驱动 | `2.8 系统运行时语义/` |

### 4.3 证据文件

- `2.1 知识梳理/2.1.1 基本概念.md`
- `2.6 形式语义/2.6.1 语义定义.md`
- `2.8 系统运行时语义/2.8.1 运行时行为与调度模型.md`

### 4.4 覆盖状态

- Socket API 概念：**部分覆盖**
- epoll/kqueue 等 Linux 扩展：**未覆盖**
- 异步 I/O（POSIX aio / Linux io_uring）：**未覆盖**

### 4.5 形式化/可验证链接

- 外部参考：TLA+ QUIC/TCP 握手模型 `tools/tla-specifications/QUIC.tla`（可类比扩展 POSIX socket 状态机）。

## 5. POSIX.1-2024 §17 — 实时扩展

### 5.1 标准条款原文

POSIX.1-2024 §17（Realtime）定义实时调度、信号量、共享内存、消息队列等（paraphrase）：

> "Interfaces that support applications with deterministic response time requirements, including priority scheduling, real-time signals, semaphores, shared memory, and message queues."

### 5.2 与项目知识点的映射

| 标准接口 | 项目知识点 | 项目章节 |
|---|---|---|
| SCHED_FIFO / SCHED_RR | 实时调度策略、优先级倒置 | `2.8 系统运行时语义/` |
| POSIX semaphores / mutex | 进程/线程同步、互斥 | `2.1 知识梳理/` |
| Shared memory objects | 共享内存、地址空间映射 | `2.1 知识梳理/` |
| Message queues | 进程间通信 | `2.1 知识梳理/` |

### 5.3 证据文件

- `2.1 知识梳理/2.1.1 基本概念.md`
- `2.8 系统运行时语义/2.8.1 运行时行为与调度模型.md`

### 5.4 覆盖状态

- 实时调度策略：**未覆盖**
- POSIX 实时信号量/消息队列：**未覆盖**

### 5.5 形式化/可验证链接

- 建议：建立实时调度可调度性分析模型（EDF/RMS），并关联 UPPAAL 时间自动机。

## 6. LSB 5.0 / System V ABI — 可执行文件与启动

### 6.1 标准条款原文

LSB 5.0 Core 规定 Linux 发行版二进制接口；System V ABI x86-64 定义 ELF 文件格式、调用约定、程序加载（paraphrase）：

> "The Executable and Linkable Format (ELF) defines the structure of object files, executable files, and shared objects. The System V AMD64 ABI defines the calling convention, stack layout, and program initialization."

### 6.2 与项目知识点的映射

| 标准主题 | 项目知识点 | 项目章节 |
|---|---|---|
| ELF header / sections / segments | 可执行文件结构、程序加载 | `2.1 知识梳理/`、`2.3 形式化结构/` |
| Dynamic linking / PLT / GOT | 共享库、地址重定位 | `2.1 知识梳理/` |
| System V ABI calling convention | 函数调用、栈帧、寄存器约定 | `2.1 知识梳理/` |
| LSB init scripts / systemd unit compatibility | 系统初始化、运行时服务 | `2.8 系统运行时语义/` |

### 6.3 证据文件

- `2.1 知识梳理/2.1.1 基本概念.md`
- `2.3 形式化结构/2.3.2 结构模型.md`
- `2.8 系统运行时语义/2.8.1 运行时行为与调度模型.md`

### 6.4 覆盖状态

- ELF 基础结构：**部分覆盖**
- LSB 系统初始化规范：**未覆盖**
- ABI 调用约定细节：**未覆盖**

### 6.5 形式化/可验证链接

- 建议：使用 Isabelle/HOL 或 Lean 4 建立 ELF 程序加载与地址空间映射的小步操作语义。
