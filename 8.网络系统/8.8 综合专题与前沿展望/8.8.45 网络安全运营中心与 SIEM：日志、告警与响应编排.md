## 1. 概述与核心目标

安全运营中心（SOC, Security Operations Center）负责持续监测、检测、分析与响应网络安全事件。安全信息与事件管理（SIEM, Security Information and Event Management）系统聚合多源日志、执行关联分析并生成告警，为 SOC 提供核心数据平台。

核心目标：

- **全面可见性**：覆盖终端、网络、身份、云工作负载与业务应用。
- **实时检测**：基于规则、统计异常与威胁情报识别安全事件。
- **高效响应**：通过标准化流程与自动化编排缩短 MTTD/MTTR。
- **合规取证**：满足 NIST SP 800-61 Rev. 2 的事件处理与证据保全要求。

```text
数据源(终端/网络/云/身份)
        |
        v
日志收集 -> 解析/标准化 -> SIEM 关联引擎 -> 告警/工单
        |                                    |
        v                                    v
  长期存储/取证                      SOAR 剧本/响应编排
```

## 2. SOC 功能层级与运营流程

参考 NIST SP 800-61 Rev. 2，事件处理生命周期包括：准备、检测与分析、遏制/根除/恢复、事后活动。

| SOC 层级 | 职能 | 典型活动 |
|---|---|---|
| L1 监测 | 告警 triage、初步分类 | 确认误报、记录工单、升级事件 |
| L2 分析 | 深度调查、威胁狩猎 | 日志关联、IOC 验证、攻击路径重建 |
| L3 专家 | 高级威胁分析 | APT 追踪、规则优化、狩猎规划 |
| IR 团队 | 遏制、根除、恢复 | 隔离主机、阻断域名、补丁修复 |

关键指标：MTTD、MTTR、误报率。

## 3. SIEM 架构与数据管道

| 组件 | 职责 |
|---|---|
| 日志收集器 | 通过 Agent/Syslog/API 收集原始日志 |
| 解析器 | 将异构日志抽取为统一字段模型 |
| 关联引擎 | 基于时间窗口、统计阈值与规则生成告警 |
| 存储与索引 | 支持热/温/冷分层与合规保留周期 |
| 可视化与报告 | Dashboard、告警队列、合规报表 |

数据流示例：防火墙/EDR/云审计/AD 日志 -> 统一 ECS/CEF 模型 -> 关联规则 -> 高优先级告警 -> SOAR/IR。

## 4. 日志标准化：CEF、Sysmon 与 Sigma

| 标准/工具 | 作用 | 来源 |
|---|---|---|
| CEF | ArcSight 日志格式，含设备、签名、严重度、扩展字段 | Micro Focus |
| LEEF | IBM QRadar 日志格式 | IBM |
| Sysmon | Windows 高级监控工具，记录进程/网络/驱动/DNS | Microsoft Sysinternals |
| Sigma | 通用签名格式，YAML 表达，可转译为 SIEM/EDR 查询 | Sigma Project |

Sysmon 典型事件 ID：1 进程创建、3 网络连接、7 镜像加载、22 DNS 查询。

Sigma 规则示例：

```yaml
title: Suspicious PowerShell Download
logsource:
  category: process_creation
  product: windows
detection:
  selection:
    CommandLine|contains:
      - 'IEX(New-Object Net.WebClient)'
  condition: selection
```

## 5. 告警关联与 MITRE ATT&CK 映射

将 SIEM 规则映射到 MITRE ATT&CK 技术 ID，可量化覆盖缺口并指导狩猎优先级。

```text
检测规则库
    |
    v
MITRE ATT&CK 技术映射(T1059.001, T1071...)
    |
    v
覆盖缺口报告 -> 新增规则 / 增强数据源
```

关联分析技术：基于规则的关联、统计异常/UEBA、威胁情报匹配、多阶段攻击链分析。

## 6. SOAR 与事件响应编排

安全编排、自动化与响应（SOAR）连接人员、流程与工具：

| 能力 | 说明 |
|---|---|
| 编排 | 连接 SIEM、EDR、防火墙、邮件网关、ITSM |
| 自动化 | 自动隔离、阻断、禁用账户、收集证据 |
| 响应 | 事件升级、通知、工单、合规报告 |

典型 playbook：SIEM 触发恶意 IP 外联告警 -> SOAR 查询情报置信度 -> 高置信度则阻断/隔离/工单，低置信度则分析师复核。

## 7. 工程实现要点

- **数据源优先级**：优先接入身份、终端、网络边界、云审计等高价值日志。
- **字段标准化**：采用 ECS 或 CEF，减少跨源关联成本。
- **分层存储**：热数据 SSD 索引、温数据压缩对象存储、冷数据归档。
- **高可用与弹性**：采集链路需支持断点续传、背压控制与多活部署。
- **访问控制**：SOC 人员权限按角色隔离，日志本身成为高价值攻击目标。

## 8. 安全分析与批判性分析

| 风险 | 说明 | 缓解 |
|---|---|---|
| 告警疲劳 | 海量低置信度告警淹没关键事件 | 优先级评分、ML 降噪、ATT&CK 聚焦 |
| 数据孤岛 | 云、OT、SaaS 日志未统一接入 | 标准化 API、统一数据湖 |
| 规则绕过 | 攻击者针对 Sigma/SIEM 规则做 OPSEC | 行为检测、威胁狩猎、红队验证 |
| 自动化误伤 | 自动隔离可能中断关键业务 | 人机协同确认、白名单、演练 |
| 隐私与合规 | 日志含个人数据，跨境存储受限 | 数据分级、匿名化、属地化存储 |

- **SIEM 不是银弹**：数据源缺失或日志质量低下无法靠关联引擎弥补；ATT&CK 覆盖率高也不等于检测有效，需红队/紫队演练验证。
- **运营文化重于工具**：SOC 效能取决于流程、人员技能与跨部门协作。
