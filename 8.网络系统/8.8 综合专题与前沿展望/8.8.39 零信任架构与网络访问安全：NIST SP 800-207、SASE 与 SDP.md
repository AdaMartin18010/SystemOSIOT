## 1. 概述与核心原则

零信任（Zero Trust）摒弃“内网可信、边界防护”的传统假设，核心信条为 **“永不信任，持续验证”**。NIST SP 800-207 将其归纳为：

- 所有数据源与计算服务均被视为资源；
- 无论网络位置如何，所有通信必须受保护；
- 按会话授予最小权限访问；
- 访问决策基于动态策略与多属性信任评估；
- 持续监控并测量资产完整性与安全态势；
- 动态严格认证与授权。

```text
传统模型：              零信任模型：
[内网] = 可信           [任何位置] = 不可信
    |                       |
边界防火墙              PDP(策略决策点) + PEP(策略执行点)
    |                       |
VPN 接入                设备态/身份态/行为态持续评估
```

## 2. NIST SP 800-207 零信任架构

[NIST SP 800-207](https://csrc.nist.gov/publications/detail/sp/800/207/final) 定义逻辑组件：

| 组件 | 职责 |
|---|---|
| Policy Engine (PE) | 最终访问决策，基于信任算法与策略规则 |
| Policy Administrator (PA) | 建立/切断会话，生成认证令牌与访问凭证 |
| Policy Enforcement Point (PEP) | 在实际资源前执行允许/拒绝/监控 |
| Continuous Diagnostics and Mitigation (CDM) | 实时资产状态与威胁情报 |
| Identity Credential and Access Management (ICAM) | 身份生命周期与多因素认证 |

## 3. SDP 软件定义边界

CSA SDP Architecture Guide v2.0 将 SDP 设计为**黑暗云（black cloud）**：

- **Initiating Host (IH)**：用户/设备端代理。
- **Accepting Host (AH)**：受保护资源前的网关。
- **SDP Controller**：认证 IH、分发 AH 可达性策略。

```text
IH ---(mTLS + 设备认证)--> SDP Controller
                              |
                    验证通过后下发 SPA / ACL
                              |
IH ---(单包授权 SPA)--> AH ---(代理)--> 内部资源
```

单包授权（SPA）使 AH 在网络层默认不可见，降低扫描与暴力破解面。

## 4. SASE 框架

Gartner 提出的 SASE（Secure Access Service Edge）将网络与安全能力融合为云服务：

| 能力 | 说明 |
|---|---|
| SD-WAN | 广域网优化与动态选路 |
| SSE（Security Service Edge） | SWG、CASB、ZTNA、FWaaS、RBI 等 |
| ZTNA | 基于身份的细粒度应用访问 |
| Cloud-Native | 以 PoP 为单位全球分布部署 |

SASE 与 SDP/ZTNA 的关系：ZTNA 是 SASE 的访问控制子集；SDP 是一种 ZTNA 实现架构。

## 5. 身份与访问管理

| 标准/协议 | 作用 |
|---|---|
| [NIST SP 800-63](https://csrc.nist.gov/publications/detail/sp/800/63/final) | 数字身份指南：IAL/AAL/FAL 三等级 |
| [RFC 6749](https://datatracker.ietf.org/doc/html/rfc6749) | OAuth 2.0 授权框架 |
| [OpenID Connect](https://openid.net/developers/specs/) | 基于 OAuth 2.0 的身份层 |

```text
用户 -> IdP（OIDC/OAuth） -> 颁发访问令牌
    |
PEP 校验令牌 + 设备证书 + 风险评分
    |
PDP 实时决策：允许 / 拒绝 / 降级 / MFA
```

## 6. 工程实践

- **微分段**：将网络划分为细粒度安全区，东西向流量默认拒绝。
- **设备信任**：TPM/TEE 证明、MDM 注册状态、漏洞评分。
- **条件访问**：位置、时间、异常行为、威胁情报触发动态策略。
- **BeyondCorp / Microsoft Azure AD / Okta**：主流商业实现参考。
- **开源组件**：Keycloak/Authentik、OpenZiti、Teleport、HashiCorp Boundary。

## 7. 批判性分析

- **复杂度与可用性**：持续 MFA、设备证书、微分段增加用户摩擦与运维负担。
- **身份系统成为新单点故障**：IdP 被攻破将导致全组织访问控制失效。
- **可见性与隐私张力**：深度终端检测与行为分析可能侵犯员工隐私，需合规平衡。
- **SASE 供应商锁定**：云原生 SASE 将网络与安全栈绑定到少数厂商，迁移成本高。
- **“零信任”营销泛化**：部分产品将传统 VPN 包装为零信任，缺乏动态信任评估与最小权限。
- **后量子准备**：TLS/mTLS 证书与身份令牌未来需迁移到抗量子算法。
