## 1. 概述与演进背景

确定性 IP（Deterministic IP, DIP）与 SRv6+ 旨在为 5G/工业互联网提供可预期延迟、抖动与丢包的网络服务。核心驱动力：

- **SRv6**：利用 IPv6 扩展头部实现源路由与网络编程，简化 MPLS 标签栈。
- **网络切片**：在同一物理基础设施上隔离出不同 SLA 的逻辑网络。
- **随流检测**：在数据包路径中嵌入测量信息，实现逐跳/逐流可视化。
- **AP6（Advanced IPv6+）**：中国 IETF 贡献者推动的 IPv6+ 增强方向，包括随流检测、网络切片、确定性 IP、应用感知等。

## 2. SRv6 网络编程与 SRH 压缩

SRv6 Network Programming（RFC 8986）定义了 Segment Routing Header（SRH）中 Segment List 的语义：

```text
IPv6 基本头部 -> 扩展头部链 -> SRH -> Segment List[0..n]
                              -> Segments Left
                              -> Segment ID (SID) = 128-bit IPv6 地址
```

关键概念：

| 概念 | 说明 |
|---|---|
| SRH | IPv6 路由扩展头部，携带 Segment List |
| SID | 128-bit IPv6 地址，可编码 locator/function/argument |
| END | 基本 Endpoint 行为：更新 DA 并转发 |
| END.X | 交叉连接行为：转发到特定邻居 |
| END.T | 表查询行为：基于 SID 查找特定转发表 |
| END.DT4/DT6 | 解封装并查找 IPv4/IPv6 路由表 |

SRH 压缩问题：128-bit SID 列表导致头部开销大，尤其在物联网/移动场景中。draft-ietf-spring-srv6-srh-compression 提出压缩 SID 列表（C-SID）与通用 SRH 压缩机制，减少 MTU 压力与芯片处理负担。

## 3. 确定性 IP 与确定性网络

确定性网络（DetNet, RFC 8938/8939/9023）提供 bounded latency、低抖动与零拥塞丢包，依赖：

- **资源预留**：为特定流预留带宽与缓存。
- **显式路由**：避免动态 IGP 收敛带来的不确定性。
- **时间同步**：IEEE 802.1AS / PTP 提供亚微秒级同步。
- **流量整形**：IEEE 802.1Qbv/Qbu 的时门调度与帧抢占。

确定性 IP 在 IP/MPLS 层借鉴 DetNet 思想，通过 SRv6 Policy 与 Flex-Algo 实现端到端确定性路径。

## 4. 网络切片：5G 与承载网协同

3GPP TS 23.501 定义 5G 网络切片，切片标识为 S-NSSAI（Single Network Slice Selection Assistance Information）：

```text
UE --(RRC/NAS)--> gNB --(NGAP)--> AMF/SMF/UPF
                        |
                        v
                   Transport Network Slice
                        |
                        v
                   SRv6 Policy / VPN / VLAN / FlexE
```

承载网切片技术：

| 技术 | 隔离粒度 | 适用场景 |
|---|---|---|
| FlexE | PHY/MAC 硬切片，时隙级隔离 | eMBB/uRLLC/mMTC 严格隔离 |
| VLAN/VxLAN | L2/L3 软切片 | 企业 VPN、边缘计算 |
| SRv6 Policy | 基于 Color 的路径差异化 | 同一网络中区分 SLA |
| VPN+ | BGP VPN 增强 | 切片级业务路由 |

## 5. 随流检测与性能测量

随流检测（iFIT, in-situ Flow Information Telemetry；IPPM 领域也称 IOAM/INT 思路）在数据包中嵌入测量元数据：

| 机制 | 描述 | 来源 |
|---|---|---|
| IOAM | In-situ OAM，在 IPv6 选项或 SRH 中携带追踪数据 | RFC 9197 / RFC 9484 |
| Alternate Marking | 交替标记法测量丢包与延迟 | RFC 8321 |
| TWAMP / STAMP | 主动测量协议 | RFC 5357 / RFC 8762 |

iFIT/AP6 中常见的测量字段包括：入/出接口、时间戳、队列深度、转发延迟、丢包标记。

## 6. AP6：IPv6 演进与可编程测量

AP6（Advanced Programmable IPv6+）是中国推动的 IPv6+ 技术集，重点包括：

- **随流检测（IFIT）**：逐包/逐流可视化，支持微突发检测。
- **网络切片（Slice）**：基于 SRv6 Policy 与 FlexE 的端到端切片。
- **确定性 IP**：为工业互联网提供确定性转发。
- **应用感知（APN6）**：在 IPv6 数据面携带应用标识，实现精细化调度。

ITU-T Y.3800 系列则定义了 IMT-2020/5G 传输网络要求，支持切片、同步与确定性。

## 7. 工程实现与部署模式

- **控制器**：SRv6 Policy 通常由 BGP-LS/PCEP/BGP SR Policy 下发。
- **芯片支持**：需 NPU/ASIC 支持 SRH 处理、压缩与 iFIT 封装。
- **互操作**：多厂商场景需统一 SRv6 SID 规划、BGP LU/Link-State 扩展。
- **运维可视化**：结合 BGP-LS 拓扑与 iFIT 数据构建数字孪生网络。

## 8. 批判性分析

- **SRH 开销与 MTU**：128-bit SID 列表增加包长，需路径 MTU 发现或压缩机制。
- **压缩复杂度**：SRH 压缩虽减少带宽，但增加芯片解析复杂度与调试难度。
- **切片隔离粒度**：软切片共享物理资源，uRLLC 场景仍需 FlexE 等硬切片补充。
- **数据面可编程与安全**：iFIT/APN6 在包中携带额外元数据，可能被滥用或泄露拓扑。
- **标准化分裂**：SRv6+、AP6、DetNet、ITU-T Y.3800 等术语与边界尚未完全统一。
- **部署成本**：端到端升级路由器、控制器、OSS 与芯片生态，投资回报周期长。
