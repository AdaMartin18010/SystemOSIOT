## 1. 数据中心网络虚拟化需求

现代数据中心需要：

- **多租户隔离**：同一物理网络承载多个虚拟网络（VPC/VNet）。
- **二层扩展**：虚拟机/容器跨机架迁移时保持 IP/MAC 不变。
- **地址空间扩展**：VLAN 4094 个 ID 限制无法满足大规模云环境。
- **与编排系统集成**：Kubernetes、OpenStack 等需要标准化网络接口。

## 2. VXLAN

VXLAN（Virtual Extensible LAN）在 [RFC 7348](https://datatracker.ietf.org/doc/html/rfc7348) 中定义，将二层帧封装在 UDP/4789 中，使用 24bit VNI 标识租户，支持约 1600 万个隔离段。

```text
[ Outer MAC ] [ Outer IP/UDP ] [ VXLAN Header (VNI) ] [ Inner Ethernet Frame ]
```

关键组件：

| 组件 | 说明 |
|---|---|
| VTEP（VXLAN Tunnel Endpoint） | 封装/解封装 VXLAN 的隧道端点 |
| VNI（VXLAN Network Identifier） | 24bit 租户/网络标识 |

来源：[RFC 7348](https://datatracker.ietf.org/doc/html/rfc7348)。

## 3. Geneve

Geneve 在 [RFC 8926](https://datatracker.ietf.org/doc/html/rfc8926) 中定义，是 NVGRE/VXLAN 的演进型 Overlay 协议，同样基于 UDP，但引入可扩展的 Type-Length-Value（TLV）选项。

```text
[ Outer MAC ] [ Outer IP/UDP ] [ Geneve Base Header ] [ Options TLVs ] [ Inner Ethernet Frame ]
```

Geneve 特点：

- 24bit VNI。
- 可扩展选项，支持元数据传递（如安全组、QoS、追踪）。
- 被 Linux 内核、Open vSwitch、VMware NSX 等广泛支持。

## 4. VXLAN vs Geneve

| 特性 | VXLAN | Geneve |
|---|---|---|
| RFC | [RFC 7348](https://datatracker.ietf.org/doc/html/rfc7348) | [RFC 8926](https://datatracker.ietf.org/doc/html/rfc8926) |
| 端口 | UDP/4789 | UDP/6081 |
| 可扩展性 | 固定头部 | TLV 选项 |
| 生态 | Cisco、VMware、Linux、Kubernetes | Linux、OVS、VMware NSX、Azure |
| 典型场景 | 通用数据中心 Overlay | 需要扩展元数据的云网络 |

## 5. CNI：容器网络接口

CNI（Container Network Interface）是 CNCF 项目，为容器运行时提供标准化网络配置接口：

- 规范：[CNI Spec](https://www.cni.dev/docs/spec/)
- 插件：bridge、host-local、macvlan、ipvlan、flannel、Calico、Cilium、Weave 等。

CNI 工作流程：

```text
容器运行时（kubelet/containerd）
   |
   | 调用 CNI 插件（ADD/DEL/CHECK）
   v
CNI 插件配置 veth/bridge/VXLAN/Geneve/IPAM
   |
   v
容器获得 IP 并接入 Pod/Service 网络
```

CNI 插件常与 VXLAN/Geneve/IP-in-IP 隧道结合，实现跨节点容器网络。

## 6. SR-IOV 与硬件卸载

SR-IOV（Single Root I/O Virtualization）由 PCI-SIG 定义，允许单个物理 NIC 虚拟出多个 VF（Virtual Function），直接分配给虚拟机/容器，降低 hypervisor 网络栈开销。

硬件卸载技术：

- **VXLAN/Geneve offload**：NIC 直接封装/解封装隧道报文。
- **eBPF/XDP**：Linux 内核可编程数据平面，用于高性能容器网络策略。

来源：[PCI-SIG SR-IOV](https://pcisig.com/specifications)、Linux `docs.kernel.org/networking/xdp.html`。

## 7. 工程实践

- **Kubernetes 网络**：Calico（BGP/IP-in-IP）、Cilium（eBPF）、Flannel（VXLAN）是主流 CNI 实现。
- **OpenStack Neutron**：ML2 插件 + OVS 实现 VXLAN/Geneve Overlay。
- **云厂商**：AWS VPC、Azure VNet、GCP VPC 均基于 Overlay + 软件定义网络实现。

## 8. 批判性分析

- **Overlay vs Underlay 可见性**：Overlay 隐藏了虚拟机/容器的真实流量模式，给网络排障与可观测性带来挑战。
- **MTU 问题**：VXLAN/Geneve 封装增加 50+ 字节开销，需调整 Underlay MTU 以避免分片。
- **CNI 碎片化**：各 CNI 插件功能与数据平面差异大，跨平台迁移和策略一致性难以保证。
