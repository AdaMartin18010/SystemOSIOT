## 1. SDN 与可编程网络概述

软件定义网络（SDN）将控制平面与数据平面解耦，使网络行为可通过集中控制器编程。可编程网络进一步允许数据包转发逻辑本身由程序员定义，而非固化在交换机 ASIC 中。

| 层次 | 功能 | 代表技术 |
|---|---|---|
| 应用层 | 网络应用（负载均衡、防火墙、流量工程） | ONOS/ODL 应用 |
| 控制层 | 网络状态聚合、路由计算、流表下发 | OpenFlow 控制器、P4Runtime 服务器 |
| 数据层 | 高速包转发、匹配-动作处理 | OpenFlow 交换机、P4 目标设备 |

## 2. OpenFlow 协议

OpenFlow 由 ONF（Open Networking Foundation）定义，最新规范为 1.5.1。它通过流表（Flow Table）实现匹配-动作（Match-Action）转发。

OpenFlow 流表项：

```text
匹配字段（Match Fields） + 优先级（Priority） + 计数器（Counters）
+ 指令（Instructions） + 超时（Timeouts） + Cookie
```

匹配字段包括：入端口、源/目的 MAC、源/目的 IP、TCP/UDP 端口、VLAN ID、MPLS 标签、隧道 ID 等。

动作包括：转发到端口、丢弃、修改字段、推/弹标签、封装到控制器等。

来源：[ONF OpenFlow 1.5.1 Specification](https://opennetworking.org/software-defined-standards/)。

## 3. P4 语言

P4（Programming Protocol-independent Packet Processors）是描述数据平面包处理行为的领域特定语言，当前版本 P4-16。

P4 程序核心抽象：

| 抽象 | 说明 |
|---|---|
| Parser | 定义如何从比特流提取头部 |
| Match-Action Tables | 基于提取的头部字段执行查找与动作 |
| Control Flow | 定义 tables 的调用顺序与条件逻辑 |
| Deparser | 将处理后的头部重新序列化为出站包 |

示例结构：

```text
P4-16 Program
├── Parser
│   └── extract(ethernet, ipv4, tcp/udp)
├── Ingress Control
│   ├── table ipv4_forward
│   └── apply(ipv4_forward)
├── Egress Control
│   └── optional modifications
└── Deparser
    └── emit(headers)
```

来源：[P4 Language Specification](https://p4.org/specs/)、[P4-16 Language Specification v1.2.4](https://p4.org/p4-spec/docs/P4-16-v1.2.4.html)。

## 4. P4Runtime 与控制平面

P4Runtime 是控制平面与数据平面之间的标准 API，基于 gRPC，定义于 [P4Runtime Specification](https://p4.org/specs/)。

- 允许运行时插入、修改、删除表项。
- 支持Packet I/O：控制器接收/注入数据包。
- 与目标设备上加载的 P4 程序信息模型（P4Info）配合工作。

## 5. 数据平面可编程硬件

| 平台 | 特点 |
|---|---|
| 软件交换机 | BMv2 (Behavioral Model)、P4C 编译目标 | 用于原型与测试 |
| SmartNIC / DPU | NVIDIA BlueField、Intel IPU、AMD Pensando | 主机卸载、可编程卸载 |
| 可编程交换机 ASIC | Intel Tofino、Broadcom Trident3/Trident4、Cisco Silicon One | Tbps 级可编程转发 |
| FPGA | Xilinx Alveo、Intel Stratix | 灵活但功耗/延迟较高 |

## 6. 典型应用场景

- **网络遥测**：INT（In-band Network Telemetry），实时测量每包延迟、队列深度。
- **负载均衡**：P4 实现一致性哈希、连接跟踪、DDoS 缓解。
- **网络功能虚拟化（NFV）**：在可编程交换机上实现防火墙、NAT、VPN 网关。
- **数据中心网络**：自定义拥塞控制信号、RoCEv2 优化。

## 7. 形式化语义与研究前沿

P4 的形式化语义是活跃研究方向：

- **P4Cub / Petr4**：Coq 中的 P4 形式化语义项目，定义语法、类型系统与可操作语义。
- **p4v**：基于符号执行的 P4 数据平面验证工具。
- **p4c 验证后端**：利用 SMT/Z3 检查表项一致性与可达性。

## 8. 批判性分析

- **可移植性**：P4 程序依赖目标设备的架构模型（PSA、v1model、TNA），跨硬件移植仍需人工调整。
- **调试难度**：数据平面并行流水线与表项交互复杂，传统抓包难以定位。
- **安全风险**：控制器集中化与可编程性扩大攻击面，需结合零信任与细粒度权限控制。
