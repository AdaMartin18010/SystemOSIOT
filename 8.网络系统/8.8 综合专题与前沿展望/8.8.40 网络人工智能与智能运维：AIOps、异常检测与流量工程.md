## 1. 概述与 AIOps 能力域

AIOps（Artificial Intelligence for IT Operations）将大数据与机器学习引入网络运维，核心能力包括**观测、分析、决策、执行**四阶段闭环。Gartner 将 AIOps 平台定义为同时 ingestion 多种数据源并提供实时分析与主动行动的解决方案。

```text
数据摄入（logs/metrics/traces/topology）
    |
    v
模式发现与异常检测
    |
    v
根因定位与事件关联
    |
    v
自动化修复 / 策略推荐
```

## 2. 数据基础与遥测

AIOps 数据通常来自：

| 数据源 | 内容 | 典型协议/格式 |
|---|---|---|
| Metrics | 时序指标：带宽、延迟、丢包、CPU | Prometheus、SNMP、gNMI |
| Logs | 系统日志、流日志、应用日志 | syslog、JSON、CEF |
| Traces | 分布式请求路径与延迟 | OpenTelemetry、Jaeger |
| Topology | 网络/服务依赖关系 | LLDP、BGP-LS、CMDB |

数据治理要点：采样率、标签一致性、时区对齐、PII 脱敏、冷热分层。

## 3. 异常检测方法

| 方法 | 适用场景 | 局限 |
|---|---|---|
| 统计阈值 / 3σ | 简单指标突变 | 对周期性与噪声敏感 |
| 时间序列预测（ARIMA/Prophet） | 容量规划、趋势预测 | 需平稳或显式季节性 |
| 无监督聚类（K-Means/DBSCAN） | 未知异常模式发现 | 高维解释性差 |
| 图神经网络（GNN） | 拓扑感知的根因定位 | 训练数据与拓扑标注成本高 |
| 深度学习（LSTM/VAE/Transformer） | 多变量序列异常 | 需要大量标注与算力 |

概念上可参考 ACM CIKM/KDD 在异常检测、图挖掘与日志模式识别方向的研究成果。

## 4. 智能流量工程

传统流量工程依赖静态 MPLS-TE 或 RSVP-TE，智能流量工程引入实时数据驱动决策：

```text
Telemetry 采集 -> 流量矩阵估计 -> ML 预测 -> 路径优化 -> BGP/SDN 下发
    ^                                                           |
    |<---------------- 效果反馈与策略修正 <----------------------|
```

关键技术：

- **BGP FlowSpec v2（[RFC 9234](https://datatracker.ietf.org/doc/html/rfc9234)）**：通过 BGP 分发流量过滤与重定向规则，用于 DDoS 缓解。
- **SRv6 + 控制器**：基于实时延迟/利用率动态调整 Segment List。
- **强化学习（RL）**：在数据中心网络中动态调整拥塞控制参数或负载均衡策略。

## 5. 工程实现与运营框架

| 框架/标准 | 作用 |
|---|---|
| ITIL 4 | 服务管理实践，强调价值流与持续改进 |
| MEF 3.0 | 运营商级 SD-WAN/SASE 服务编排与 SLA |
| IEEE 802.1Qcc | TSN 集中式网络配置，也可纳入 AIOps 控制闭环 |
| OpenTelemetry | 可观测数据标准化与互操作 |

典型平台栈：

- 数据湖：Kafka + ClickHouse / Druid
- 特征工程：Spark / Flink
- 模型服务：MLflow / Seldon
- 执行编排：Ansible / Salt / Nornir / Kubernetes Operator

MEF 3.0 在运营商场景下定义了 SD-WAN、SASE 等服务的标准化编排接口，使 AIOps 平台能够以统一方式管理多厂商 Underlay/Overlay 与 SLA 合约。

## 6. AIOps 实施路线图

```text
阶段 1：可观测性统一（metrics/logs/traces 接入）
阶段 2：告警降噪与事件关联（去重、抑制、拓扑聚合）
阶段 3：预测性分析（容量预测、异常检测、变更影响评估）
阶段 4：半自动修复（Runbook 自动化、灰度回滚）
阶段 5：闭环自治（策略自优化、人机协同审批）
```

每阶段需配套数据治理、SRE 成熟度评估与回退机制，避免过早引入自动化导致故障半径扩大。

## 7. 批判性分析

- **数据质量决定上限**：告警标签缺失、重复事件、配置漂移会导致模型失效。
- **可解释性不足**：深度学习模型给出的异常评分难以被网络工程师采信，需结合因果推理与知识图谱。
- **反馈闭环风险**：自动修复动作可能放大故障（如误关闭 BGP 会话），需人工审批与灰度策略。
- **组织变革挑战**：AIOps 不仅是技术平台，更需要 SRE/NOC/DevOps 流程重塑。
- **能耗与碳足迹**：大规模模型训练与实时推理增加数据中心能耗，与绿色网络目标存在张力。
