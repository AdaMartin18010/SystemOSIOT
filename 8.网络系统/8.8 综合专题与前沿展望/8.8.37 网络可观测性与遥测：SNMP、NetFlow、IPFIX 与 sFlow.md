## 1. 概述与可观测性分层

网络可观测性通过多源遥测数据回答“发生了什么、为什么发生、如何影响业务”。核心 telemetry 来源可分为三层：

```text
基础设施层：设备 CPU/内存/接口状态（SNMP）
    |
流量层：五元组/应用识别/字节/包统计（NetFlow/IPFIX/sFlow）
    |
事件层：系统日志、Trap、审计记录（syslog/SNMP Trap）
    |
汇聚层：SIEM / NPM / AIOps 平台
```

三类数据互为补充：SNMP 适合状态轮询，流技术适合流量画像，syslog 适合事件关联。

## 2. SNMPv3 架构与安全

SNMPv3 在 [RFC 3411](https://datatracker.ietf.org/doc/html/rfc3411)-3418 中定义，重点解决 SNMPv1/v2c 的明文 community string 问题：

| 组件 | 功能 | 权威来源 |
|---|---|---|
| SNMP-FRAMEWORK-MIB | 管理 SNMP 引擎标识与架构 | [RFC 3411](https://datatracker.ietf.org/doc/html/rfc3411) |
| SNMP-MPD / USM | 消息处理模型与用户安全模型 | [RFC 3412](https://datatracker.ietf.org/doc/html/rfc3412)、[RFC 3414](https://datatracker.ietf.org/doc/html/rfc3414) |
| SNMP-VACM | 基于视图的访问控制模型 | [RFC 3415](https://datatracker.ietf.org/doc/html/rfc3415) |
| Target/Notify | 通知目标与过滤器配置 | [RFC 3413](https://datatracker.ietf.org/doc/html/rfc3413) |

安全服务：

- **认证**：HMAC-MD5-96 或 HMAC-SHA-96（现代部署推荐 SHA）。
- **加密**：DES / AES（RFC 3414/3826）。
- **访问控制**：VACM 按 group/securityLevel/read-view/write-view 限制 OID 范围。

## 3. NetFlow 与 IPFIX 流记录

Cisco NetFlow 最初为 v5/v9，IETF 将其模板化思想标准化为 IPFIX（[RFC 7011](https://datatracker.ietf.org/doc/html/rfc7011)）。

```text
[路由器/交换机]
    | 提取流记录（5/7 元组 + 字节/包/时间戳）
    v
[NetFlow/IPFIX 收集器]
    | 模板解析 -> 数据记录存储
    v
[分析平台] 流量 topN、DDoS 检测、计费
```

NetFlow v9 / IPFIX 关键改进：

| 特性 | NetFlow v5 | NetFlow v9 / IPFIX |
|---|---|---|
| 字段固定 | 固定字段 | 基于模板，可扩展 |
| IPv6 | 不支持 | 支持 |
| MPLS 标签 | 有限 | 可通过模板携带 |
| 标准化 | Cisco 私有 | IPFIX 为 IETF 标准 |

常见收集器：nfdump / nfsen、Elatic Flow、GoFlow2、Kentik。

## 4. sFlow 采样机制

sFlow（[sFlow v5](https://sflow.org/sflow_version_5.txt)）采用**统计采样**而非流记录：

- **Packet-based sampling**：每 N 个包采样一个报文头，适用于流量矩阵与异常检测。
- **Counter sampling**：周期性轮询接口计数器，开销低。

与 NetFlow/IPFIX 对比：

| 维度 | NetFlow/IPFIX | sFlow |
|---|---|---|
| 数据粒度 | 聚合流记录 | 采样原始包头 |
| CPU/内存 | 较高（需维护流缓存） | 较低（无状态采样） |
| 适用场景 | 计费、安全取证、长期趋势 | 实时异常、微突发、拓扑发现 |
| 协议来源 | IETF / Cisco | sflow.org 行业规范 |

## 5. syslog 与统一遥测管道

syslog 在 [RFC 5424](https://datatracker.ietf.org/doc/html/rfc5424) 中定义结构化日志格式：

```text
<PRI>VERSION TIMESTAMP HOSTNAME APP-NAME PROCID MSGID STRUCTURED-DATA MSG
```

典型集成架构：

```text
网络设备 / 服务器 / 应用
    | syslog / SNMP Trap / streaming telemetry (gNMI)
    v
日志代理（rsyslog / fluent-bit / vector）
    | 解析、过滤、富化
    v
消息队列（Kafka）-> 存储（Loki/Elasticsearch）-> 告警/可视化
```

现代网络正从 SNMP 轮询转向 Model-Driven Telemetry（gNMI、YANG Push），但 SNMP 与 syslog 仍是存量设备的主流接口。

## 6. 工程实现与对比

| 技术 | 部署位置 | 关键指标 | 典型工具 |
|---|---|---|---|
| SNMP | 路由器、交换机、打印机、UPS | CPU、内存、接口 up/down、错误计数 | Zabbix、Prometheus SNMP exporter、LibreNMS |
| NetFlow/IPFIX | 核心/边界路由器、SD-WAN | 流量topN、AS路径、应用识别 | nfdump、Elatic Flow、Kentik |
| sFlow | 数据中心 ToR 交换机、校园网 | 微突发、延迟异常、环路 | sFlow-RT、Ntopng |
| syslog | 全量基础设施 | 事件关联、合规审计、故障追踪 | ELK、Splunk、Grafana Loki |

## 7. 安全与批判性分析

- **SNMPv1/v2c 遗留风险**：大量设备仍使用默认 community string，是横向移动与信息收集的常见入口。
- **流数据隐私**：NetFlow/IPFIX 记录可能包含源/目的 IP、端口甚至部分 payload 信息，需脱敏与留存策略。
- **采样偏差**：sFlow 采样可能遗漏低速率但高风险的扫描流量；NetFlow 长连接超时设置不当会扭曲统计。
- **数据爆炸**：全流记录存储成本高，需在精度、保留期与成本之间权衡。
- **多源关联复杂度**：SNMP、syslog、flow、tracing 时间同步与字段对齐是工程落地难点，依赖 NTP/PTP 与统一 schema。
