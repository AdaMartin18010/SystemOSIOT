## 1. 概述与系统分类

非地面网络（NTN, Non-Terrestrial Network）泛指利用卫星、高空平台（HAPS）、无人机等空中/空间节点提供连接的网络系统。卫星互联网是其最重要形态，按轨道高度分为：

| 类型 | 高度 | 延迟（单程） | 典型系统 |
|---|---|---|---|
| LEO 低轨 | 300–2,000 km | ~2–7 ms | Starlink、OneWeb、Kuiper、Iridium |
| MEO 中轨 | 8,000–20,000 km | ~30–60 ms | O3b/mPOWER |
| GEO 地球静止轨道 | ~35,786 km | ~120 ms | Viasat、Inmarsat、HughesNet |
| HAPS | 20–50 km | <1 ms | 高空伪卫星、 stratospheric balloons |

NTN 与 5G/6G 融合目标：扩展覆盖至海洋、航空、偏远地区与应急场景。

## 2. NTN 信道特征与传输挑战

```text
地面终端 <-> 卫星/高空平台 <-> 信关站 <-> 核心网/互联网
```

关键物理层挑战：

- **大传播时延**：GEO 往返可达 500 ms 以上，LEO 虽低但动态变化。
- **多普勒频移**：LEO 高速运动导致显著频偏，需补偿。
- **高路径损耗**：自由空间损耗随距离平方增长，需高增益波束与链路预算。
- **遮蔽与衰落**：雨衰、大气吸收、建筑/植被遮挡影响可用性。
- **星地动态切换**：LEO 星座需频繁切换波束与卫星，影响移动性管理。

## 3. 3GPP NR-NTN 架构与协议适配

3GPP 在 Release 17/18 引入 NR-NTN，将 5G NR 扩展到卫星场景：

| 文档 | 内容 |
|---|---|
| TR 38.811 | NR 支持 NTN 的研究：场景、信道模型、部署选项 |
| TR 38.821 | NR-NTN 解决方案：RAN 架构、物理层、协议增强 |
| TR 22.822 | 5G 系统 NTN 用例与需求 |
| TS 38.101-5 | NTN 终端射频与性能要求 |

关键协议增强：

- **定时提前（TA）扩展**：补偿长往返时延。
- **HARQ 重传自适应**：避免长 RTT 导致过多 HARQ 进程阻塞。
- **多普勒补偿**：通过预补偿与快速频率同步。
- **SIB19 / SIB20**：广播 NTN 辅助信息（星历、参考 TAI 时间、公共定时提前）。
- **移动性管理**：基于地球固定波束（Earth-fixed beam）或地球移动波束（Earth-moving beam）的切换策略。

## 4. Starlink 与低轨星座工程实践

Starlink 是当前规模最大的商业 LEO 星座，工程特点：

- **星座规模**：数千颗卫星，采用 Walker 星座部署，实现全球覆盖。
- **相控阵天线**：用户终端使用电控相控阵，无需机械跟踪。
- **星间激光链路（ISL）**：部分卫星支持激光通信，减少地面网关依赖。
- **自主碰撞规避**：基于公开轨道数据与星载推进系统。
- **运营挑战**：频谱协调、太空碎片、光污染、监管许可。

| 维度 | Starlink 工程取舍 |
|---|---|
| 延迟 | LEO 优势，但切换与排队引入抖动 |
| 容量 | 频率复用与波束成形提升，但密集区仍受限 |
| 韧性 | 分布式星座抗单点故障，但网关与核心网仍是地面依赖 |
| 监管 | 需逐国频谱与落地权谈判 |

## 5. DVB-RCS2 与卫星宽带回传

DVB-RCS2（Digital Video Broadcasting — Return Channel via Satellite 2）是 ETSI 主导的卫星交互标准，广泛用于宽带卫星回传：

- **前向链路**：DVB-S2/S2X 高速广播。
- **返向链路**：MF-TDMA 多址接入，支持 QoS 与自适应编码调制。
- **应用场景**：农村宽带、海事、航空、企业 VPN 备份。

与 3GPP NR-NTN 相比，DVB-RCS2 更偏向广播与固定/游牧宽带，而 NR-NTN 强调与蜂窝生态融合。

## 6. IP over Satellite 与传输层优化

卫星链路的长时延、高误码与不对称带宽对 TCP/IP 影响显著：

| 问题 | 影响 | 工程缓解 |
|---|---|---|
| TCP 慢启动在长 RTT 下吞吐低 | 带宽时延积（BDP）大，窗口增长慢 | 加大初始拥塞窗口、BBR、PEP 性能增强代理 |
| 误码触发 TCP 拥塞误判 | 卫星误码并非网络拥塞 | 链路层 ARQ/FEC、SNACK、TCP-split PEP |
| 不对称路由 | 返向容量有限 | 头压缩、ACK 过滤与聚合 |

IETF 在 RFC 3137 等文档中讨论了慢启动对长时延链路的影响；现代部署常结合 Split-TCP、SCPS-TP 或 QUIC 以改善体验。

## 7. 批判性分析

- **太空碎片与可持续性**：大规模 LEO 星座增加碰撞风险，需主动离轨与碎片监测。
- **数字鸿沟 vs 天文观测**：卫星互联网可弥合接入鸿沟，但卫星反光干扰天文观测。
- **主权与监管**：卫星信号跨境覆盖挑战传统电信主权与频谱分配。
- **地面基础设施依赖**：多数系统仍需地面网关接入互联网，战时或灾害中可能失效。
- **成本与能效**：星座建设、发射与运维成本高昂，终端功耗与价格限制普及。
- **与地面 5G 融合复杂性**：NR-NTN 需解决双连接、计费、紧急呼叫、漫游等运营商级问题。
