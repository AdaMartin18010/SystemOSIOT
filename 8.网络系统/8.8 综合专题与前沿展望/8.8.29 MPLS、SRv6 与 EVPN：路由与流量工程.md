## 1. MPLS 基础架构

多协议标签交换（Multiprotocol Label Switching, MPLS）在 [RFC 3031](https://datatracker.ietf.org/doc/html/rfc3031) 中定义，位于 OSI 第 2.5 层，在数据链路层与网络层之间插入固定长度标签，实现快速转发。

```text
IP 包
  |
  v
[ MPLS 标签栈 ]  -- 标签值 20bit, EXP 3bit, S 1bit, TTL 8bit
  |
  v
数据链路层帧
```

核心组件：

| 组件 | 说明 |
|---|---|
| LER（Label Edge Router） | 入口/出口标签边缘路由器 |
| LSR（Label Switch Router） | 核心标签交换路由器 |
| FEC（Forwarding Equivalence Class） | 具有相同转发处理方式的数据流分类 |
| NHLFE（Next Hop Label Forwarding Entry） | 下一跳标签转发表项 |

## 2. 标签分发与 LSP

标签分发协议包括：

- **LDP**（Label Distribution Protocol）：[RFC 5036](https://datatracker.ietf.org/doc/html/rfc5036)，基于 IGP 拓扑分发标签。
- **RSVP-TE**：[RFC 3209](https://datatracker.ietf.org/doc/html/rfc3209)，支持显式路径与带宽预留的流量工程扩展。
- **BGP**：[RFC 8277](https://datatracker.ietf.org/doc/html/rfc8277) 用于 VPNv4/VPNv6 标签分发。

标签交换路径（LSP）是 MPLS 数据包从入口 LER 到出口 LER 的转发路径，可通过 IGP 最短路径或 RSVP-TE 显式路径建立。

## 3. 流量工程（TE）与 RSVP-TE

MPLS-TE 允许网络管理员：

- 绕开拥塞链路，优化带宽利用。
- 建立具有带宽保证的显式路径。
- 实现快速重路由（FRR）以提升可靠性。

RSVP-TE 扩展了 RSVP，用于在 MPLS 网络中建立带约束的 LSP，约束条件包括带宽、链路属性、管理组等。

来源：[RFC 3209](https://datatracker.ietf.org/doc/html/rfc3209)、[RFC 4090](https://datatracker.ietf.org/doc/html/rfc4090)（Fast Reroute）。

## 4. Segment Routing 与 SRv6

Segment Routing（SR）是一种源路由技术，由 [RFC 8402](https://datatracker.ietf.org/doc/html/rfc8402) 定义，将路径编码为 Segment 列表。

| 类型 | 数据平面 | 关键 RFC |
|---|---|---|
| SR-MPLS | MPLS 标签栈 | [RFC 8402](https://datatracker.ietf.org/doc/html/rfc8402) |
| SRv6 | IPv6 扩展头 Segment Routing Header（SRH） | [RFC 8986](https://datatracker.ietf.org/doc/html/rfc8986) |

SRv6 优势：

- 无需 MPLS 数据平面，直接利用 IPv6 转发平面。
- 通过 SRH 中的 Segment List 实现源路由、VPN、流量工程。
- 简化控制平面，可通过 BGP/IGP 分发 Segment 信息。

## 5. EVPN

EVPN（Ethernet VPN）在 [RFC 7432](https://datatracker.ietf.org/doc/html/rfc7432) 中定义，基于 BGP MPLS 提供以太网服务，后续扩展支持 SRv6（[RFC 8365](https://datatracker.ietf.org/doc/html/rfc8365) 等）。

EVPN 路由类型：

| 类型 | 名称 | 用途 |
|---|---|---|
| 1 | Ethernet Auto-Discovery | 别名、多宿主 |
| 2 | MAC/IP Advertisement | MAC 与 ARP/ND 同步 |
| 3 | Inclusive Multicast Ethernet Tag | 广播/未知单播/组播转发 |
| 4 | Ethernet Segment Route | 多宿主冗余 |
| 5 | IP Prefix Route | 跨子网 IP 前缀通告 |

EVPN 广泛应用于数据中心 VXLAN/BGP-EVPN 架构与运营商以太网服务。

## 6. 对比与选型

| 技术 | 控制平面 | 数据平面 | 主要场景 |
|---|---|---|---|
| MPLS + LDP/RSVP-TE | LDP / RSVP-TE | MPLS 标签 | 运营商骨干、VPN、TE |
| SR-MPLS | IGP/BGP | MPLS 标签 | 简化 TE、大规模骨干 |
| SRv6 | IGP/BGP | IPv6 + SRH | 5G 承载网、云骨干 |
| EVPN | BGP | MPLS / VXLAN / SRv6 | 数据中心互联、L2/L3 VPN |

## 7. 工程实现

- **路由器/交换机**：Cisco IOS-XR、Juniper JunOS、Arista EOS、Nokia SR Linux 均支持 MPLS/SR/EVPN。
- **Linux**：FRRouting 支持 BGP-EVPN、SRv6；Linux 内核支持 MPLS 转发与 SRv6 路由。
- **SDN 控制器**：ONOS、OpenDaylight 提供 SR/EVPN 编排能力。

## 8. 批判性分析

- **协议栈复杂度**：MPLS/SRv6/EVPN 控制平面涉及 IGP、BGP、PCEP 等多种协议，运维难度高。
- **SRv6 报头开销**：SRH 增加 IPv6 扩展头长度，对小包场景影响显著。
- **互通性与迁移**：从 MPLS 向 SRv6 迁移需考虑设备能力、控制平面共存与流量工程一致性。
