## 1. 概述与确定性需求

传统以太网基于统计复用与尽力而为转发，无法满足工业控制、车载网络、音视频传输、移动前传等场景对**bounded latency、低抖动、高可靠性** 的需求。TSN（Time-Sensitive Networking）与 DetNet（Deterministic Networking）分别从二层与三层提供确定性保证。

```text
确定性需求来源
    |-- 工业自动化（PLC、机器人同步）
    |-- 汽车以太网（ADAS、传感器融合）
    |-- 专业音视频（AES67 / SMPTE ST 2110）
    |-- 移动前传（eCPRI / 5G fronthaul）
```

## 2. IEEE 802.1AS gPTP 时间同步

[IEEE 802.1AS-2020](https://standards.ieee.org/standard/802.1AS-2020.html) 基于 IEEE 1588 PTP，但针对桥接局域网做了简化和Profile：

| 特性 | IEEE 1588-2008 | IEEE 802.1AS gPTP |
|---|---|---|
| 传输层 | UDP/IP 或二层 | 仅二层（Ethertype 0x88F7） |
| 时钟选择 | BMC 算法 | 精简后的 gPTP 状态机 |
| 透明时钟 | 可选 | 桥必须支持透明时钟修正 |
| 精度目标 | 亚微秒级 | 通常 < 1 μs（局域网） |

gPTP 关键消息：

- **Sync + Follow_Up**：主时钟向下游分发时间。
- **Pdelay_Req / Pdelay_Resp / Pdelay_Resp_Follow_Up**：测量链路传播延迟。

```text
Grandmaster
    | Sync
    v
Bridge (透明时钟修正 residence time)
    | Sync + Follow_Up
    v
Slave Clock
```

## 3. TSN 流量整形与调度

TSN 通过 IEEE 802.1Q 系列扩展实现时隙、抢占与冗余：

| 标准 | 机制 | 作用 |
|---|---|---|
| [IEEE 802.1Qbv-2015](https://standards.ieee.org/standard/802.1Qbv-2015.html) | 增强流量整形（TAS） | 门控调度，按时间片开放队列 |
| [IEEE 802.1Qbu-2016](https://standards.ieee.org/standard/802.1Qbu-2016.html) | 帧抢占 | 高优先级帧可中断低优先级帧 |
| [IEEE 802.1Qca-2015](https://standards.ieee.org/standard/802.1Qca-2015.html) | 路径控制与预留 | 基于 ISIS 的显式路径与冗余 |
| IEEE 802.1CB | 帧复制与消除 | 无缝冗余，容忍单点故障 |
| IEEE 802.1Qcc | 集中式网络配置 | CNC 与 CUC 接口（YANG 数据模型） |

802.1Qbv TAS 示意：

```text
时间轴：|---- T1 ----|---- T2 ----|---- T3 ----|
门状态：  [Q7-Q0 开]  [仅 Q7-Q5 开] [Q7-Q0 开]
          关键流量      音视频/控制       尽力而为
```

802.1Qbu 帧抢占将低优先级帧拆分为 fragment，使时间关键帧无需等待长帧发送完毕。

## 4. IETF DetNet 确定性网络

DetNet 在三层网络（IP/MPLS）上提供确定性转发，核心 RFC 包括：

| RFC | 内容 |
|---|---|
| [RFC 8938](https://datatracker.ietf.org/doc/html/rfc8938) | DetNet 数据平面框架 |
| [RFC 8939](https://datatracker.ietf.org/doc/html/rfc8939) | DetNet IP 数据平面 |
| [RFC 9023](https://datatracker.ietf.org/doc/html/rfc9023) | DetNet MPLS 数据平面 |
| [RFC 9055](https://datatracker.ietf.org/doc/html/rfc9055) | DetNet 安全考量 |

DetNet 关键机制：

- **Explicit Route**：通过 RSVP-TE/SR 预留端到端路径。
- **Packet Replication and Elimination (PREOF)**：复制流并在接收端消除重复，提高可靠性。
- **Flow Aggregation**：将多个 DetNet 流聚合为一条隧道，降低状态开销。
- **Service Protection**：1+1 或 1:1 冗余，配合丢包/乱序恢复。

## 5. 工程部署模型

```text
现场层（TSN）              汇聚/骨干层（DetNet）
PLC / 机械臂                  IP/MPLS 路由器
    |--- 802.1AS gPTP ---|        |--- RSVP-TE/SR ---|
    |--- 802.1Qbv TAS  ---|        |--- PREOF 冗余 ---|
    TSN 交换机                   DetNet 边界节点
```

典型厂商栈：

- TSN：Intel i210/i225 TSN 网卡、TSNswitch、Cisco IE 9300。
- DetNet：Juniper MX/PTX、Cisco ASR 9000（实验/部分实现）。
- 配置：YANG/NETCONF（IEEE 802.1Qcc）、OPC UA TSN 映射。

## 6. 局限性与批判性分析

- **配置复杂度**：TAS 门控列表需要精确计算传输时间、帧长度与拓扑延迟，出错即导致确定性破坏。
- **异构互操作**：不同厂商 TSN 实现细节差异大，集中配置（CNC/CUC）仍处早期标准化阶段。
- **规模限制**：DetNet 预留状态随流数量线性增长，大规模互联网核心部署仍受限。
- **成本与生态**：TSN 网卡与交换机价格高于普通以太网，工业现场存量 PROFINET/EtherCAT 迁移成本高。
- **测试挑战**：亚微秒级同步与纳秒级抖动需要专用测试仪（如 Ixia、Spirent），CI/CD 难以完全自动化。
