## 1. 概述与演进背景

SD-WAN（Software-Defined Wide Area Network）通过软件定义方式简化分支机构互联，支持基于应用策略的多链路选路。SASE（Secure Access Service Edge）由 Gartner 提出，将 SD-WAN 与云原生安全服务（SSE）融合为统一边缘交付平台。

演进路径：

```text
传统 MPLS VPN -> 混合 WAN(MPLS + 互联网) -> SD-WAN(应用感知选路)
       -> SASE(SD-WAN + 云安全服务 + 零信任访问)
```

核心驱动力：云化与 SaaS 普及、远程办公常态化、安全能力云化、网络可编程化。

## 2. SD-WAN 服务属性：MEF 70 与 MEF 3.0

MEF 70 定义 SD-WAN 服务的服务属性与信息模型：

| 服务属性 | 说明 |
|---|---|
| SD-WAN UNI | 用户与 SD-WAN 服务的边界 |
| SD-WAN 虚拟连接 | UNI 之间应用感知逻辑连接 |
| 应用识别与策略 | 基于五元组、DPI、FQDN 识别应用并映射 Underlay |
| 性能监控 | SLA 指标：延迟、抖动、丢包、带宽 |
| 安全服务 | IPsec、防火墙、分段等可叠加属性 |

MEF 3.0 强调自动化服务编排（LSO API）、多云连接与标准化 SLA。

## 3. SD-WAN 控制平面与数据平面

```text
分支 CPE/Edge
    |
    +-- Underlay 1: MPLS
    +-- Underlay 2: 宽带互联网
    +-- Underlay 3: 4G/5G
    |
    v
SD-WAN Controller(策略/编排/监控)
    |
    v
数据中心 / 云 / SaaS
```

| 平面 | 技术 |
|---|---|
| 控制平面 | SD-WAN Controller、BGP、OVMP、NETCONF/RESTCONF |
| 数据平面 | IPsec/GRE/DTLS/VXLAN Overlay、SRv6、应用感知转发 |
| 管理平面 | Orchestrator、Telemetry、AIOps、ZTP |

应用感知选路示例：语音/视频会议优先低延迟链路；文件传输优先高带宽低成本链路；SaaS 采用本地互联网分流。

## 4. SASE 框架与核心能力

Gartner SASE 将网络与安全能力融合为分布式云服务：

| 能力 | 说明 |
|---|---|
| SD-WAN | 广域网优化与应用感知选路 |
| SSE | 云安全服务集合 |
| ZTNA | 零信任网络访问 |
| SWG | 安全 Web 网关 |
| CASB | 云访问安全代理 |
| FWaaS | 防火墙即服务 |
| RBI | 远程浏览器隔离 |

部署形态：

```text
用户/分支 -> PoP 边缘 -> SASE 云服务 -> 企业资源/SaaS/IaaS
              |
        SD-WAN + SSE 融合处理
```

## 5. 零信任与 SASE：NIST SP 800-207

NIST SP 800-207 的零信任原则与 SASE 天然契合：永不信任、持续验证；最小权限；按会话动态授权；持续监测资产完整性。

在 SASE 中，零信任通过身份上下文、设备信任、应用级访问与持续评估实现。

## 6. SRv6 与 SD-WAN/SASE 融合

SRv6（RFC 8986）通过 IPv6 扩展头部实现源路由，可简化 SD-WAN Overlay 并支持精细流量工程。

```text
SRv6 SID 规划示例:
  2001:db8:1:A::/64  -> 数据中心 A
  2001:db8:1:B::/64  -> 云 POP B
  2001:db8:1:C::/64  -> SASE 安全服务链 C
```

融合价值：简化封装、服务链串联安全能力、跨域统一流量工程。

## 7. 工程实现要点

- **PoP 选址**：靠近用户与云区域，兼顾合规与数据主权。
- **身份架构**：统一 IdP（OIDC/SAML/LDAP）是零信任与 SASE 基础。
- **Underlay 多样性**：至少 MPLS + 互联网 + 4G/5G，避免单点故障。
- **DPI 与隐私平衡**：深度包检测提升应用识别精度，但需合规处理用户数据。
- **ZTP 与配置治理**：分支机构设备自动上线，配置变更需版本控制与回滚。
- **可观测性**：统一采集 Underlay/Overlay 性能、安全事件、用户行为。
- **主流方案**：Cisco Viptela、VMware VeloCloud、Fortinet、Palo Alto Prisma、Cloudflare One、Zscaler。

## 8. 批判性分析

- **供应商锁定**：SASE 将网络与安全栈深度绑定到云服务商，迁移与互操作困难。
- **性能与延迟**：所有流量经过 PoP 安全检测可能增加时延，尤其音视频场景。
- **隐私与合规**：用户流量在第三方 PoP 解密检查，可能触碰 GDPR/数据本地化要求。
- **复杂度转移**：虽然简化分支运维，但核心架构与策略治理复杂度上升。
- **零信任落地难度**：身份、设备、应用、数据全面改造周期长，易出现"半零信任"状态。
- **SRv6 部署门槛**：需要全网 IPv6 支持、芯片升级与跨厂商互操作。
