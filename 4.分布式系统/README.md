## 核心概念

### 4.0.1 分布式系统定义

分布式系统（Distributed System）是由多个独立计算机组成的系统，这些计算机通过网络进行通信和协调，共同完成特定任务。

### 4.0.2 系统特征

- **并发性**：多个进程同时执行
- **缺乏全局时钟**：各节点时钟不同步
- **故障独立性**：单个节点故障不影响整体
- **消息传递**：通过消息进行通信

### 4.0.3 系统架构

```text
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   节点A         │    │   节点B         │    │   节点C         │
│ ┌─────────────┐ │    │ ┌─────────────┐ │    │ ┌─────────────┐ │
│ │  应用层     │ │    │ │  应用层     │ │    │ │  应用层     │ │
│ ├─────────────┤ │    │ ├─────────────┤ │    │ ├─────────────┤ │
│ │  中间件层   │ │◄──►│ │  中间件层   │ │◄──►│ │  中间件层   │ │
│ ├─────────────┤ │    │ ├─────────────┤ │    │ ├─────────────┤ │
│ │  网络层     │ │    │ │  网络层     │ │    │ │  网络层     │ │
│ └─────────────┘ │    │ └─────────────┘ │    │ └─────────────┘ │
└─────────────────┘    └─────────────────┘    └─────────────────┘
```

### 4.0.4 核心问题

- **一致性**：CAP定理、ACID、BASE
- **可用性**：故障检测、故障恢复、容错
- **分区容错性**：网络分区处理
- **共识算法**：Paxos、Raft、拜占庭容错
