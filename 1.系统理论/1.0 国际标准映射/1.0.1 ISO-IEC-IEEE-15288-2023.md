## 3. 关键缺口优先级

| 优先级 | 缺口 | 建议补齐内容 |
|---|---|---|
| P0 | 生命周期过程与七维框架未显式映射 | 在 `1.1 知识梳理` 中新增“15288 生命周期视角下的系统理论”章节 |
| P0 | 验证/确认/证明概念混淆 | 在 `1.4 形式化证明` 中严格区分 Verification vs Validation vs Proof |
| P1 | 运行时语义覆盖不足 | 在 `1.8 系统运行时语义` 中按 15288 技术过程组织运行时模型 |
| P1 | 架构描述未对齐 ISO 42010 | 在 `1.5 多表征` 中引入 ISO/IEC/IEEE 42010:2022 视图与视点 |
| P2 | 组织/协议过程缺失 | 在 `1.2 批判分析` 中增加系统作为合同/组织 artifact 的批判视角 |

## 4. 与 SysML v2 / KerML 的协同

ISO/IEC/IEEE 15288:2023 强调基于模型的系统工程（MBSE）。SysML v2 / KerML v1.0 可作为项目多表征层（`1.5`）和形式化结构层（`1.3`）的标准建模语言。建议：

- 使用 SysML v2 的 `Concern`、`Viewpoint`、`View` 重新组织 `1.5 多表征`。
- 使用 KerML 的 `Type`、`Feature`、`Connection` 形式化系统要素与关系。
