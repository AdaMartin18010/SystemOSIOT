## 3. KerML v1.0 形式化基础

KerML 是 SysML v2 的底层内核语言，提供：

- **Type/Feature**：类型与特征，可用于形式化系统要素。
- **Connection**：连接，可用于形式化系统关系。
- **Succession**：时序，可用于形式化系统演化/运行时语义。
- **Behavior/Function**：行为/函数，可用于形式化系统功能。

建议将项目中的“系统 = (要素, 关系, 边界, 功能)”定义与 KerML 的 Type-Feature-Connection 框架对齐。

## 4. 多表征层升级建议

当前 `1.5 多表征` 主要使用 Mermaid 图。建议按 SysML v2 标准视图补充：

| 视图类型 | SysML v2 工件 | 项目落地文件 |
|---|---|---|
| 结构视图 | Block Definition Diagram / Interconnection Diagram | `1.5/结构图.md` |
| 行为视图 | Activity Diagram / State Machine Diagram | `1.6/语义模型.md` |
| 需求视图 | Requirements Diagram | `1.3/形式化定义.md` |
| 参数视图 | Parametric Diagram | `1.4/命题与定理.md` |
| 架构视图 | View / Viewpoint | 新增 `1.5/标准视图.md` |

## 5. 与 Coq 形式化的衔接

`tools/coq-verification/SystemTheory.v` 中定义了 `Element`、`Relation`、`System` 等归纳类型。建议：

- 在注释中说明这些类型对应 KerML 的 `Feature` / `Connection` / `Type` 概念。
- 明确 `System` Record 与 SysML v2 `Part` / `Item` 的映射关系。
- 对 `system_hierarchy`、`system_emergence` 等定理补充 SysML v2 视图解释。
