## 1. 标准概述

| 属性 | 内容 |
|---|---|
| 标准族 | BitTorrent Enhancement Proposals (BEPs) |
| 核心规范 | BEP 3 (Peer Wire Protocol)、BEP 5 (DHT Protocol)、BEP 44 (DHT 任意数据存储) |
| DHT 理论 | Kademlia: A Peer-to-Peer Information System Based on the XOR Metric (Maymounkov & Mazières, LNCS 2002) |
| 现代 P2P 栈 | libp2p、IPFS |
| 官方链接 | BEP Index: <https://www.bittorrent.org/beps/bep_0000.html>；Kademlia paper: <https://pdos.csail.mit.edu/~petar/papers/maymounkov-kademlia-lncs.pdf>；libp2p specs: <https://specs.libp2p.io/>；IPFS specs: <https://specs.ipfs.tech/> |

## 2. 标准条款原文

### 2.1 BitTorrent BEP 3 — Peer Wire Protocol

> The peer protocol operates over TCP or uTP. … Peers exchange a handshake, followed by a never-ending stream of length-prefixed messages. The messages are: choke, unchoke, interested, not interested, have, bitfield, request, piece, cancel, port.
> — BitTorrent.org, BEP 3

关键条款：

- 握手：`pstrlen`、`pstr`、`reserved`、`info_hash`、`peer_id`。
- 阻塞算法（Choking）：基于当前下载速率选择 unchoke 同伴； optimistic unchoke 用于发现新同伴。
- 片段请求：`request <index><begin><length>`，通常 length = 2¹⁴ bytes。

### 2.2 BitTorrent BEP 5 — DHT Protocol

> The DHT is composed of nodes, each of which has a globally unique identifier called the node ID. Node IDs are selected such that they are close to the infohash of the torrents they are interested in.
> — BitTorrent.org, BEP 5

关键条款：

- KRPC 消息：查询（q）与响应（r），字段 `t`（transaction ID）、`y`（类型）、`q`（查询类型）、`a`（参数）、`r`（响应）。
- 四种查询：`ping`、`find_node`、`get_peers`、`announce_peer`。
- 路由表维护：基于 node ID 的 XOR 距离分层（k-buckets）。

### 2.3 Kademlia 论文核心条款

> The Kademlia routing algorithm uses XOR-based metric topology. For any given node, the routing table is organized into a series of k-buckets, each covering a portion of the 160-bit ID space.
> — Maymounkov & Mazières, 2002

关键结论：

- XOR 距离满足非负性、对称性、三角不等式。
- 查找复杂度：O(log n) 跳，消息复杂度：O(log n)。
- 通过并发、异步 RPC 提升查找效率。

### 2.4 libp2p / IPFS 关键条款

- libp2p `multiaddr`：自描述网络地址，例如 `/ip4/1.2.3.4/tcp/4001/p2p/QmPeerID`。
- IPFS CID（Content Identifier）：自描述内容哈希，支持 CIDv0（base58btc multihash）与 CIDv1（multibase + multicodec + multihash）。
- IPFS Bitswap：基于 want-list 的块交换协议，激励兼容通过账本（ledger）记账。
