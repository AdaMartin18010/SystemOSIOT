## 标准条款原文

### MPI 4.0 核心条款

MPI（Message-Passing Interface）4.0 标准由 MPI Forum 于 2021-06-09 发布，官方文档为：
<https://www.mpi-forum.org/docs/mpi-4.0/mpi40-report.pdf>

与本项目直接相关的条款包括：

| 条款 | 标题 | 核心内容摘要 |
|---|---|---|
| 第 3 章 | Point-to-Point Communication | 定义标准、同步/就绪/缓冲发送模式，以及 `MPI_Send`、`MPI_Recv`、`MPI_Isend`、`MPI_Irecv` 等原语的语义与正确性约束。 |
| 第 5 章 | Collective Communication | 定义 `MPI_Bcast`、`MPI_Reduce`、`MPI_Allreduce`、`MPI_Gather`、`MPI_Scatter`、`MPI_Barrier` 等集合操作，要求所有参与进程调用匹配的集合例程。 |
| 第 7 章 | Process Topologies | 定义笛卡尔拓扑（`MPI_Cart_create`）与图拓扑（`MPI_Graph_create`），支持进程到物理处理器的映射。 |
| 第 11 章 | One-Sided Communication | 定义 MPI RMA（Remote Memory Access）模型，包括窗口（Window）、同步机制（Active/Passive Target）、原子操作与内存一致性规则。 |
| 第 12 章 | External Interfaces | 定义 MPI 与外部工具/运行时的接口（如 `MPI_Comm_spawn`、`MPI_Publish_name`）。 |

### OpenMP 5.2 / 6.0 核心条款

OpenMP API 官方规范地址：
<https://www.openmp.org/specifications/>

| 条款 | 标题 | 核心内容摘要 |
|---|---|---|
| 第 10 章 | Parallel Regions | `parallel` 构造、线程团队（thread team）及数据环境规则。 |
| 第 11 章 | Worksharing-Loop Constructs | `for` / `do` / `loop` 指令、`schedule` 子句、`simd` 与 `distribute` 的调度语义。 |
| 第 12 章 | Tasking | `task` / `taskloop` / `taskwait` / `taskgroup` 任务模型与依赖。 |
| 第 13 章 | Device Offloading | `target` 构造、设备数据环境及 `map` 子句，适用于加速器/异构节点。 |

### HPCG / Graph500 基准

- HPCG（High Performance Conjugate Gradients）Benchmark：
  <https://www.hpcg-benchmark.org/>
  - 核心指标：加权 GFLOP/s，基于预处理共轭梯度（PCG）迭代阶段的操作与耗时。
  - 有效运行要求：官方运行时间 ≥ 1800 秒，问题规模需占用主内存的显著部分（通常 ≥ 1/4）。

- Graph500 Benchmark：
  <https://graph500.org/>
  - 核心指标：BFS/SSSP 的 median TEPS（Traversed Edges Per Second）。
  - 重点考察大规模图遍历对通信子系统的压力。

## 覆盖状态

- **MPI 点对点通信**：部分覆盖。`5.1.1` 已定义集群节点与负载均衡，但未显式引入 MPI 消息传递语义。
- **MPI 集合通信**：未覆盖。`5.3/5.6` 中缺少 `Bcast/Reduce/Barrier` 等形式化描述。
- **MPI 进程拓扑**：未覆盖。`5.1.1` 的集群拓扑为通用主从/对等架构，未映射到 MPI 笛卡尔/图拓扑。
- **MPI One-Sided Communication / RMA**：未覆盖。
- **OpenMP 共享内存并行**：未覆盖。项目当前以集群级调度为主，缺少节点内 OpenMP 并行语义。
- **HPCG / Graph500 基准**：未覆盖。`5.5` 多表征中尚未建立基准指标体系。
