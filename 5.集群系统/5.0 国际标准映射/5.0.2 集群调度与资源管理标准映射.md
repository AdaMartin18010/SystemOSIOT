## 标准条款原文

### Slurm Workload Manager

Slurm 官方文档：<https://slurm.schedmd.com/>

| 概念/条款 | 来源/章节 | 核心内容摘要 |
|---|---|---|
| Partitions | `slurm.conf` 中 `PartitionName` | 将集群节点划分为不同队列，配置允许的用户、时间限制、优先级等。 |
| Job Allocation | `salloc`、`sbatch`、`srun` 文档 | 为作业分配独占或非独占节点访问权，并启动、执行、监控并行作业。 |
| Scheduling Plugins | `sched/wiki` / `sched/backfill` | 支持多种调度插件，包括 Backfill（回填调度）、Priority（优先级）、Topology 等。 |
| Preemption / QoS | `preempt.html` / `qos.html` | 支持基于 QoS 的作业抢占、资源预留与服务质量等级。 |
| Generic Resources (GRES) | `gres.html` | 支持 GPU、FPGA 等异构资源（`--gres=gpu:2`）的分配与记账。 |
| Energy/Power Management | `power_mgmt.html` / `acct_gather_energy` | 支持节点能耗采集与频率/功耗上限管理。 |

### OGF JSDL v1.0 (GFD.56)

官方文档：<https://ogf.org/documents/GFD.56.pdf>

> "The Job Submission Description Language (JSDL) is an XML language used to describe the submission requirements of computational jobs to resources in Grid environments."

核心元素：

- `JobDescription`：作业描述根元素。
- `JobIdentification`：作业名称、描述、项目。
- `Application`：可执行文件、参数、输入/输出。
- `Resources`：CPU 数量、物理内存、运行时间、操作系统、候选主机。
- `DataStaging`：输入数据上传与输出数据下载。

### OGF DRMAA v2.2

官方文档：<https://www.drmaa.org/documents.php>

| 条款 | 核心内容摘要 |
|---|---|
| Job Session | 会话抽象，用于组织作业提交与管理。 |
| Job Template | 描述作业属性：远程命令、参数、资源需求、环境变量、工作目录。 |
| Job Submission | `run`、`runBulk` 等非阻塞提交接口。 |
| Job State Model | 作业状态机（Queued、Running、Suspended、Done、Failed 等）。 |
| Job Control | 终止、挂起、恢复、信号发送等控制操作。 |

### Kubernetes v1.33 调度

官方文档：<https://v1-33.docs.kubernetes.io/docs/concepts/scheduling-eviction/>

| 概念/条款 | 核心内容摘要 |
|---|---|
| kube-scheduler | 默认调度器，负责为未调度 Pod 选择最佳节点。 |
| Scheduling Framework | v1 调度框架，定义 Filter、Score、Reserve、Permit、Bind 等扩展点。 |
| Pod Scheduling Gates | 允许外部控制器控制 Pod 调度时机。 |
| Resource Requests/Limits | CPU/内存请求与上限，影响调度与 QoS 等级（Guaranteed/Burstable/BestEffort）。 |
| Affinity / Anti-affinity / Taints & Tolerations | 节点亲和性、Pod 间亲和/反亲和、污点与容忍。 |

### POSIX.1 (IEEE Std 1003.1)

官方文档：<https://pubs.opengroup.org/onlinepubs/9799919799/>

与集群调度相关的 POSIX 接口包括：

- 进程控制：`fork`、`exec`、`exit`、`waitpid`
- 资源限制：`getrlimit` / `setrlimit`
- 实时调度：`sched_setscheduler`、`sched_setparam`
- 信号与进程组：`kill`、`setpgid`、`setsid`

### ETSI MEC GS MEC 003

官方页面：<https://www.etsi.org/technologies/multi-access-edge-computing>

> "MEC offers application developers and content providers cloud-computing capabilities and an IT service environment at the edge of the network."（ETSI MEC 技术简介）

MEC 003 定义：

- MEC 系统参考架构（MEC platform、MEC management、MEC host）。
- 应用生命周期管理（APP 实例化、终止、迁移）。
- 网络信息暴露（RNIS、Location Service、Bandwidth Management）。

## 覆盖状态

- **Slurm 队列与 QoS**：部分覆盖。`5.1.1` 的调度器定义与 `5.7.1` 的工程案例已提及 Slurm，但未细化到 Partitions、Preemption、Backfill 策略。
- **Slurm GRES / 能耗管理**：未覆盖。项目缺少 GPU/FPGA 异构资源分配与能耗管理章节。
- **JSDL / DRMAA**：未覆盖。项目当前未涉及网格计算时代的作业描述语言与统一 API。
- **Kubernetes 调度框架**：部分覆盖。`5.7.1` 已列出 Kubernetes 案例，但未系统讲解 Scheduling Framework 扩展点、Pod Scheduling Gates、QoS。
- **POSIX 资源限制与实时调度**：部分覆盖。`5.7.2` 涉及运行时机制，但未与 POSIX 接口逐项对照。
- **ETSI MEC 边缘集群**：未覆盖。项目未涉及边缘计算场景下的集群架构与应用生命周期管理。
